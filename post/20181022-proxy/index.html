动态代理

 发表于 2018-10-22  |  4 分钟 (744 个字)

设计模式更多的是一种语意，而不是一种语法；

一、了解代理

问题：1.要知道一个方法的运行时间？（不更改源码）

@1继承，用子类代理父类的方式实现。

@2实现，聚合方式实现代理。

问题：2.需要进行多种操作，在计算时间时，同时添加日志、事务…、…操作？
@1继承实现代理的方式,臃肿、繁琐
  需要无限向下创建子类进行代理。
@2聚合方式，更加便捷方便。

二、了解动态代理

静态代理： 代理类事先自己定义好，程序运行前编译完成;
动态代理： 程序运行时创建代理类;
@1代理类的创建过程简单概述

    要实现动态代理，需要一个代理类的实例Proxy.newProxyInstance
    1.需要java字节码文件,根本不需要知道代理类的具体类名;
      实现动态的编译有多种方式
        a. jdk6 complier API（字符串类编译）
        b. CGLib,ASM (直接生成字节码)
    2.将java字节码文件加载到内存，并创建实例;

@2完善字符串类适应各需求

    按需求任意拼装字符串
    1.任意interface的代理：
                            传入接口,Proxy.newProxyInstance<Class interface>    
    2.implements了该interface的所有方法都代理：
                            利用反射得到该接口的所有方法,然后组织字符串;

@3完善处理方式
当前代理只能完成时间处理，因为我们的处理是写死在代码里的；
before:long start = System.currentTimeMillis();
after:long end = System.currentTimeMillis();

    解决思路：方法里before干什么，after干什么，由别人动态的指定;
             也就是说方法不用自己执行，交给别人执行，让别人去处理;

方法交给handler处理

完善上述思路，方法的实现需要得到具体对象,调整代码
就此，动态代理解析清楚了

 1public class TransactionHandler implements InvocationHandler {
 2	
 3	private Object target;
 4	
 5	public TransactionHandler(Object target) {
 6		super();
 7		this.target = target;
 8	}
 9
10	@Override
11	public void invoke(Object o, Method m) {
12		//事务开始
13		System.out.println("Transaction Start");
14
15		try {
16			m.invoke(target);
17		} catch (Exception e) {
18			e.printStackTrace();
19		}
20		// 事务结束
21		System.out.println("Transaction Commit");
22	}

24}
